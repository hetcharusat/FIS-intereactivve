Basic Concepts of InfoSec

1.1 Introduction to Information Security

Core Terminologies

  • Asset: What we are protecting.
  • Vulnerability: A flaw or weakness in security procedures, design, or implementation.
  • Threat: A potential cause of an unwanted impact to a system or organization.
  • Attack: An intentional attempt to exploit a vulnerability.

Goals of Information Security (The CIA Triad)

The foundation of all security implementations is the CIA Triad:

  • Confidentiality: Ensuring only authorized users access data.
  • Integrity: Ensuring data is not altered or corrupted.
  • Availability: Ensuring systems and data are available to authorized users when needed.

1.2 Implementation Issues of the Goals

Implementing the CIA Triad is not without challenges. In the real world, absolute security is impossible without completely isolating a system.

Standard Trade-offs

  • Security vs. Usability: The more secure a system is (e.g., forcing complex 15-character passwords changing every 30 days), the harder it is for employees to use.
  • Security vs. Performance: Adding heavy encryption (Confidentiality) slows down data transmission and processing.
  • Cost vs. Benefit: You cannot spend $100,000 to protect an asset worth $10,000. Risk assessment must justify the implementation.

1.3 Control Mechanisms for Information Security

Controls are the policies, procedures, and technologies used to mitigate risks.

Types of Controls

  • Technical (Logical): Software/hardware rules (e.g., Firewalls, Encryption, Antivirus).
  • Administrative (Managerial): Policies, guidelines, and training (e.g., Employee onboarding, password policies).
  • Physical: Real-world security (e.g., Security guards, locks, CCTV, badge readers).

Control Functions

Preventive (stops the attack), Detective (identifies the attack), Corrective (fixes the damage).

1.4 Access Control - Administrative and Technical

Access control ensures that a Subject (User/Program) can only interact with an Object (File/Database) based on their assigned permissions.

Administrative Access Control

These are the human-driven rules. Examples include: Principle of Least Privilege (giving users only the bare minimum access they need), Separation of Duties (requiring two people to complete a sensitive task), and Mandatory Vacations (to detect fraud).

Technical Access Control

How the system enforces the rules. Examples include: Access Control Lists (ACLs), Role-Based Access Control (RBAC - access based on job title), and MAC/DAC models.

1.5 Passwords - Are They Secure?

Historically, passwords have been the primary method of authentication, but their security is highly questionable today.

The Human Flaw

Passwords rely on human memory. As a result, users often:

  • Use easily guessable passwords (e.g., "password123", pet names).
  • Reuse the exact same password across multiple sites (Credential Stuffing risk).
  • Write them down on sticky notes under keyboards.

Conclusion: Passwords alone are no longer considered a fully secure security boundary. They must be augmented with length, complexity, and additional factors.

1.6 Passwords, Hash Function, Common Threats

Systems should NEVER store passwords in plain text.

Hash Functions

A hash function is a one-way mathematical algorithm that converts a password into a fixed-length string of characters (e.g., SHA-256). When you log in, the system hashes your input and compares it to the stored hash.
Note: To prevent pre-computed hash attacks, we add Salt (random data) before hashing.

Common Password Threats

  • Brute-Force Attack: Trying every single possible character combination until the password is guessed.
  • Dictionary Attack: Trying thousands of common words and phrases from a predetermined list.
  • Rainbow Table Attack: Using large databases of pre-computed hashes to reverse-engineer a stolen hash.

1.7 Multifactor Authentication – Challenges

Multifactor Authentication (MFA) requires two or more different forms of authentication. The factors are:

  1. Something you know: Password, PIN, Security Question.
  2. Something you have: Smartphone (SMS/App), Smartcard, Hardware Token (YubiKey).
  3. Something you are: Biometrics (Fingerprint, Iris scan, Voice).

Challenges of Implementation

  • High Cost & Maintenance: Hardware tokens cost money; SMS gateways charge per text.
  • User Resistance: Users find the extra steps annoying, leading to friction.
  • Vulnerabilities: SMS codes can be intercepted via SIM Swap attacks; biometric data cannot be changed if compromised.

1.8 Spheres of Information Security

Security is not just an IT problem; it is a multi-layered organizational issue. The spheres visualize how security protects Information at the core.

Management / Policy (Outer Sphere)
Networks & Systems (Middle Sphere)
Information / Data (Core)

Defense in Depth: By layering these spheres (Policy, People, and Technology), an attacker who breaches the network layer still has to bypass system controls and data encryption to reach the core information.
